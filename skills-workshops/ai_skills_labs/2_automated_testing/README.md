# AI Skills Lab 2: Automated Testing

Automated tests are important, right? Hopefully, you don't need convincing of that, but if you do just think back to the previous exercise where you refactored an obfuscated codebase _without_ the support of a test suite! After the refactor, how did you know if the program still worked?? Wouldn't it have been great if you could just run a comprehensive suite of tests which would tell you either way?

## Using AI to Write Tests

Thankfully, GitHub copilot has some neat features that we can use to configure and build out a suite of tests. This is exactly what we'll cover in this workshop.

## Exercise

1. Go back to your refactored code
2. Use GH Copilot to do the initial setup for a test suite (without any meaningful tests at first)
3. Use GH Copilot to add a single test for a specific part of the codebase
4. Use GH Copilot to setup `coverage`
5. Check your test coverage and keep adding more tests until coverage is at 100%

> Don't commit any code you don't understand

## Code Review

Swap codebases with your pair and review their code. If there's anything you don't understand, ask them to explain! :)

# AI Skills Lab 3: From Ticket to Quality Code

In the previous sessions you used GitHub copilot to refactor and to add tests. In this session, we'll go further and you'll use GitHub copilot to test-drive new features.

## Exercise

1. Go back to your refactored (and fully tested) codebase
2. Add a blank `plan.md` inside the `.github` directory
3. Based on the relevant `_FEATURES.md` file, add details to `plan.md`
4. Use GitHub copilot to add some tests for your first new feature. They should, of course, fail.
5. Quality Assure those tests and don't move on until you understand them
6. Use GitHub copilot to implement your first feature by making those tests pass
7. Keep TDDing until all your features have been implemented

> Don't commit any code you don't understand

## Code Review

Swap codebases with your pair and review their code. If there's anything you don't understand, ask them to explain! :)

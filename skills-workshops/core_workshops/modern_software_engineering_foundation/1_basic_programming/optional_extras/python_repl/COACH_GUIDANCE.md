
## 📋 Coach Facing Guidance: Python REPL Workshop

### Session Purpose

This session is designed to re-introduce the REPL as a high-value tool for experimentation and discovery. By mid-week 1, many participants have moved into writing scripts in VS Code and may have forgotten the REPL exists or view it as a "beginner-only" tool.

### Timing

Run this mid-to-late **Week 1**. Participants should have enough Python knowledge to understand basic data types but might be struggling with the nuance of return values versus in-place changes.

### Key Facilitation Points

* **Check for Understanding:** Use the "Returning vs Printing" section to clear up misconceptions about how variables store data.
* **The List Trap:** Pay special attention to the difference between `sorted()` and `.sort()`. This is a common source of bugs in Chapter 2.
* **Interactive Loading:** Emphasise the `python3 -i` flag. Showing them they can "import" their own functions into a sandbox is often a lightbulb moment for debugging.

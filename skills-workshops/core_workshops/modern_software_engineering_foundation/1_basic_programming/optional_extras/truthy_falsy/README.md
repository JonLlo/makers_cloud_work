## Truthy, Falsy, and Other Traps

This workshop is designed to lead you directly into the "traps" — those common, counter-intuitive behaviours in Python that even experienced developers stumble over. Think of this as a safe, controlled environment to make mistakes now so you don't make them in your projects later.

**The Golden Rule:** It is **100% okay to get things wrong**. In fact, if you don't fall into the traps during this session, you aren't doing it right! We want you to make guesses, participate in the chat, and lean into the confusion.

---

### How the Session Works

This is an interactive, "gamified" session where we prioritise experimentation and enjoy surprises :)

* Your coach will **present snippets of Python code** that look straightforward but behave in surprising ways.
* You’ll be asked to **guess the output**. Don't be afraid to be the only one with a certain answer!
* Once the "trap" is sprung, we will use the Python REPL to investigate what's going on.
* We will explore different data types—strings, integers, lists, and dictionaries to see which ones are "Truthy" and which ones are "Falsy."
* We'll wrap up by looking at how understanding these quirks can actually help you write cleaner, more "Pythonic" code.

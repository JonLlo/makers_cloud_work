# Coach Guidance: Pytest Workshop

## Session Goals

The primary objective is to move participants toward technical literacy regarding test outputs. It encourages them to treat `pytest` as a collaborator rather than an obstacle.

## Timing

This session is most effective when the group has completed **Chapter 1** and is beginning **Chapter 2** of the Python Foundations material.

* It serves as a refresher on core concepts.
* It builds processes that will help with the more complex challenges in Chapter 2.

## Delivery

### Intro

Explain how useful `pytest` output is, whilst acknowledging that it can be intimidating at first.

### Demo
1. Run `pytest -x` and spend some time working through the output with the cohort
2. After you've squeezed all the information out of the test output, make the test pass
3. Recap what you did


### Main
Give learners 15-20 mins to continue with the exercise alone.

### Plenary
Regroup to go through the next few test failures, under the guidance of the cohort.

Ensure you touch on all of these...

* **Understanding Error Messages:** Learning what the output actually means rather than guessing.
* **Deciphering Symbols:** Knowing exactly what the `+` and `-` symbols mean in a diff.
* **Getting More Info:** Knowing when and how to use `-v` or `-vv` to see more information.
* **Spotting Differences:** Noticing small discrepancies between what was returned and what was expected.

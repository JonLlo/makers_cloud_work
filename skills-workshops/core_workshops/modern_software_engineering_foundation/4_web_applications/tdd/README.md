# Test Driving Web Apps

This workshop focuses on test driving features in the ['plain' starter project](https://github.com/makersacademy/web-applications-in-python-project-starter-plain).

## Learning Objectives

This workshop can potentially cover all of the learning objectives for the module.

- [ ] Explain how HTTP requests and responses work at a high level
- [ ] Implement web routes using a lightweight web framework
- [ ] Write integration tests for a web application
- [ ] Follow a debugging process for a web application

> You'll be asked to rate yourself out of 10 for these by sending a Zoom DM to your coach

## Intro (30 mins)

During the intro, your coach will...

- Diagram the app and recap the HTTP request/response cycle at a high level
- Give a quick tour of the codebase, focusing on how one specific request is handled
- Gather ideas for a few features to TDD, for example
  - Handling invalid IDs in a `GET /books/:id` request
  - Updating a book
  - Validating a book (it should have a title and author name to be saved)
- Pick one easy feature and demonstrate some TDD

## Main (30 mins)
- Your coach will suggest 2-3 features of varying complexity
- You can choose which one/s to work on, depending on what you'd like to practice / learn

## Plenary (30 mins)
- First, we'll regroup ask everyone to share how they got on using emojis or numbers
- Then, a few learners will be invited to share a slightly more detailed update
- One or two lucky people (ideally people who are a bit stuck) will be invited to share their screen and we'll mob program a solution

> At the end, you'll be asked to re-rate yourself against the learning objectives.

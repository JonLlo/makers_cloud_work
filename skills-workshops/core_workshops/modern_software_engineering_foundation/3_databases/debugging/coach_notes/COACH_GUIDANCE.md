# How to Coach Database debugging workshop

Suggested timing: Day 3/4 

## Structure:
- Quick intro (10 mins)
    - send a zipped folder of this "debugging" folder *without this coach note* to the chat
    - remind them how to set up the codebase - point at README.md.
    - explain that they'll need to run pytest and see what tests fail to get started
    -
- Have a go (15 mins)
    - no need for breakout rooms, can work individually with cameras off
      
- Demo/mini plenary (10 mins)
    - Fliers may have found all bugs by this point (if you find that to be the case, consider adding some more bugs to this repo for next time!)
    - See which ones people are stuck on, get a learner who has solved it to talk through their process - what print statements did they use, debugger perhaps, what error messages did they spot and follow?
    
- Have another go (20 mins)

- Plenary (15 mins)

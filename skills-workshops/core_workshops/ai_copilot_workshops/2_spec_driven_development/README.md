# Spec-Driven Development

## Learning Objectives

In this workshop, you will learn to...

- Use a specification to steer Copilot (i.e. apply Spec-Driven Development)

## Intro

In the last workshop, you learned about...

- Ask mode, can help you learn about a codebase, explore concepts and make plans without changing any files
- Edit mode, which is able to make changes to existing files that are in context
- Agent mode, which can autonomously carry out complex tasks, including those which require new files and / or command line tools

In this workshop, you will learn about a new process for coding with AI call Spec-Driven Development. In this process, you will create a detailed specification for your program, which an AI agent will then use as a blueprint. Used well, it can be an incredibly productive approach to programming.

What do we mean by "used well"...?

- You write high quality, detailed specifications which guide the AI to work in small steps
- You commit regularly and often so you can easily revert to a previous state
- You take time to understand the code which is generated
- You review the generated code and provide the AI with feedback
- The generated code adheres to general principles of engineering quality

# Coach Instructions

How to run this workshop.

## Prep

- Read the top level README and the language-specific README you'll use today
- Practice spec-driven development using the approach we describe
- Fix any small mistakes/problems you encounter :)

## Schedule

- **Intro** (15 mins)
- **Main** (30 mins) 
- **Plenary** (15 mins)

## Intro

During the intro, you should:

- **Recap the previous workshop** where we covered:
  - Ask, Edit, Agent modes
  - `copilot-instructions.md`
  - Custom prompts
  - Custom chatmodes (potentially, depending on time)

- **Introduce today's focus**: "taking the brakes off" Agent mode and letting it act as an autonomous developer

- **Explain spec-driven development**:
  - The specification is a list of detailed features
  - We'll instruct the agent to work through features one by one
  - After each feature is complete, the agent stops and we perform code review
  - This is roughly comparable to test-driven development, but we only write the spec

- **Walk through the IntelliJ workaround** (for Java workshops):
  - Explain why it's necessary (IntelliJ Copilot plugin doesn't fully implement the API)
  - Demo the `copilot-instructions.md` template
  - Demo the `plan.md` template

- **Set the task**:
  - Learners should use our templates or create their own
  - Then apply spec-driven development to create a simple command line app

## Main

Allow 30 mins. You can choose whether to use pairs or individual work.

**Check in and help with**:
- Template setup issues
- Agent mode not following instructions properly
- Code review guidance
- Scope management (keeping features small)

## Plenary

1. Use Miro (or similar) to gather reflections from all learners
2. Ask a few learners to elaborate on their comments and facilitate discussion

**Key reflection questions**:
- How did spec-driven development feel compared to traditional coding?
- What worked well with the AI agent approach?
- What challenges did you encounter?
- How might you use this approach in future projects?

> NOTE: We expect there to be some variation, between learners, in terms of their experience of this exercise. Embrace that as a 'feature' of the workshop and a factor to consider when using AI coding agents.
